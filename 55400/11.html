Exercise 2 – Create calculated columns

Now that we have investigated our formatting, we now need to start finding the holes in our data. We have been asked to report on sales profit, purchase year, ages and age groups. Although we do not actually have any of this information, we do have some columns that could help us build answers. For example, our sales profit is our sales amount minus our product cost. Other examples are the purchase year which we can get from the purchase date and the age and age groups can be calculated from our customers birth date. To create these new columns, we will use DAX.

  1. To begin, let us calculate our sales profit, in the Sales table, which should be open from the previous exercise, click on the Table tools tab in the ribbon and select New column from the Calculations category as shown below

  2. In the formula bar enter SalesProfit = Sales[SalesAmount]-Sales[ProductCost] then press Enter
     
  3. Format the new column to have 2 decimal places in the Column Tools tab of the ribbon
     
  4. Next, we need our purchase year, in the Sales table, in the Table tools tab of the ribbon click on New column and enter this formula
    Purchase Year = YEAR(Sales[PurchaseDate]) then press Enter
  5. To work out the financial year of the sale, in the Table tools tab of the ribbon click on New column and enter this formula (or better yet, copy and paste it from this book):
    Financial Year = VAR fy = IF (MONTH ( Sales[PurchaseDate] ) <= 3,

VALUE ( FORMAT ( Sales[PurchaseDate], "YY" ) ) - 1,

VALUE ( FORMAT ( Sales[PurchaseDate], "YY" ) ) )

RETURN

CONCATENATE ( "FY ", CONCATENATE ( fy, CONCATENATE ("/", fy + 1 ) ) )
then press Enter
 

  1. Now we will work out our customers ages, select the Customers table from the Fields pane
     
  2. In the ribbon, in the Table tools tab, click on New column
     
  3. Enter the following into the formula bar
    Age = Floor( (TODAY()-Customers[BirthDate])/365, 1) then press Enter
     
  4. Now that we know the customers age, we can create another column to add each customer to an age group, click New column once more (from the ribbon)
     
  5. Enter the following formula (or copy and paste it from this book):
    Age Group = SWITCH( TRUE(),
    [Age] >= 65, "Ages 65 and over",
    [Age] >= 50, "Ages 50 TO 65",
    [Age] >= 40, "Ages 40 TO 49",
    [Age] >= 30, "Ages 30 TO 39",
    [Age] >= 18, "Ages 18 TO 29",
    [Age] < 18, "Ages under 18" )
     
  6. The formula should look like this if you typed it in manually, you can click in the formula and hold Shift then Enter to add return carriage to the formula (it’s not necessary but can help with the formatting)



    Below shows the formula with return carriages


     
  7. Press Enter to create the column (if you get an error, click on the column heading to access the DAX code again to make changes, remember you can always copy and paste the code from this document as long as you have it in a digital format)

Exercise 3 – Creating measures

Not all the information we calculate needs to generate a new column, for example the total sales revenue is a sum of our sales amount. In fact, to answer some questions, our calculations may need to involve data from multiple tables. This is where measures come in, allowing us to calculate information from more than one source to achieve a resultant value that can be used in our reports.

  1. We need to work out the total units sold, to do this, select the Sales table from the Fields pane (you may wish to collapse some of the expanded tables to make navigation easier)
     
  2. In the Table tools tab of the ribbon, click on New measure found in the Calculations category as shown below

  3. Enter the following into the formula bar Units Sold = SUM(Sales[Quantity])
     
  4. Press Enter on your keyboard and the new measure called Units Sold will appear under the Sales table


     
  5. To format the measure, select Units Sold from the Sales table and then click on the comma separator from the Formatting category of the ribbon and set the decimal places to 0

  6. Next, to create our sales revenue measure, in the ribbon click on New Measure from the Calculations category and enter the following into the formula bar
    Sales Revenue = Sum(Sales[SalesAmount])
     
  7. To format the measure, select Sales Revenue from the Sales table and change the type of currency by clicking on the $ drop down and select £ English (United Kingdom) and change the decimal places to 0
     
  8. Create the following measures in the Sales table using what you have practiced:
  1. Create a Product Cost measure
  1. Formula: Product Cost = Sum(Sales[ProductCost])
  2. Formatting: Currency type £ English (United Kingdom) with 0 decimal places
  1. Create a Profit measure
  1. Formula: Profit = SUM(Sales[SalesProfit])
  2. Formatting: Currency type £ English (United Kingdom) with 0 decimal places
     
  1. Now, switch over to the Customers table by selecting it from the Fields pane
     
  2. Create a new measure using the formula Customer Count = COUNTROWS(Customers)
     
  3. Format the measure by selecting Customer Count and click on the comma separator

Exercise 4 - Geographical data and lookup tables

Now that we have some new columns and measures to answer some of our grain statements, we have been asked to add more regional data to our reports. This includes which country the order is going to and what sales region is the customer based in (northern, central or southern which are boundaries devised by House Rules Board Games and are not a universally known geography). To accomplish this, we can use a lookup table provided by our HR team, which we will add to our data model. A preview of the lookup table is shown on the following page.

  1. From Model view, in the Home tab in the ribbon, select Excel workbook from the Data category
     
  2. In File Explorer, locate your training pack and open the folder Labs > Data Sources and select UK Sales Regions and click Open
     
  3. Select the UKSalesRegions table then click Load
     
  4. Click Manage relationship in the ribbon
     
  5. If a relationship is not automatically created between the UKRegionalSales and the Customers table, click Autodetect… then Close
     
  6. Select the new relationship Customers(County) to UKSalesRegions(CountyName) and click Edit
     
  7. Set the Cross filter direction to Both and click OK


     
  8. Click Close and optionally reposition the tables in your data model (Ctrl + left click to select more than one table to bulk drag)
     
  9. Navigate to Data view using the rail menu
     
  10. In the Customers table, click New Column in the Home tab of the ribbon
     
  11. Enter the following formula Country = RELATED(UKSalesRegions[Country])
     
  12. Next, click New Column in the Home tab of the ribbon
     
  13. Enter the following formula Sales Region = RELATED(UKSalesRegions[SalesRegion])
     
  14. You will now have some new geographical data for your customers, we now need to format our geographical columns
     
  15. Select the Postcode column and then in the ribbon select the Column tools tab and for Data category choose Postal code


     
  16. Repeat this step for County, adding the data category County
     
  17. Also repeat this step for Country, adding the data category Country
     
  18. Often when used on a map, some locations will be plotted in the wrong countries e.g. Athens, Greece vs Athens, Georgia, USA so to disambiguate the data we will create a final column by clicking New column from the Column tools tab
     
  19. Enter the formula Location = Customers[County] & ", " & Customers[Country]
     
  20. Press Enter on your keyboard then format the new Location column by selecting it and setting the Data Category to Place
     
  21. Lastly, we will hide the lookup table in report view as we now have the essential data in our Customers table, to do this click on Model in the rail menu
     
  22. Click on the ellipsis (…) at the top right of the UKSalesRegions table and choose Hide in report view (or click on the icon at the top right of the table)


     
  23. Click Save at the top left of the application

Exercise 5 – Create a dimensional hierarchy

We will finish up by adding a quick dimensional hierarchy to our data model. They allow us to use our columns to create a drill down through our data. We have a column for Product, Subcategory and Category which could be turned into a hierarchy to allow us to drill down through our product catalog.

  1. Click on Data view in the rail menu
     
  2. Expand the Products table from the Field pane
     
  3. To create the hierarchy, place your mouse cursor over the Category field, and then click on the ellipsis (…) then choose Create hierarchy

  4. A new hierarchy is created called Category Hierarchy
     
  5. Move your mouse cursor over the Subcategory field and click on the ellipsis (…), select Add to hierarchy > Category Hierarchy
     
  6. Repeat the previous step for the Product field
     
  7. Right click on the hierarchy heading and choose Rename
     
  8. Name the hierarchy Product Category then press Enter


     
  9. Now click Save once more at the top left of the application and you are done!
